# PR Driven Delivery Pattern

No code reaches main outside a pull request.

Every feature branch must end with

- a fully completed pull request body
- passing fast lane automated checks
- documented manual verification
- ACP artifacts updated to match reality

Use deep lane validation when the surface area or risk justifies it.

A task is not done when code exists.
A task is done when the pull request is reviewable, verifiable, and merge-safe.
